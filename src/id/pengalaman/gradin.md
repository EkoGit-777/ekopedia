# Gradin

## Tentang Perusahaan

Gradin adalah agensi digital yang berbasis di Surabaya yang percaya bahwa kreativitas dan teknologi kini tak terpisahkan. Dengan keahlian mereka dalam branding visual yang dipadukan dengan pengetahuan luas tentang web, mereka ingin membantu Anda menavigasi era digital ini untuk mengembangkan potensi sejati klien mereka.

Mereka memulai perjalanan mereka pada tahun 2010, dan kemudian memantapkan diri dengan nama GRADIN pada tahun 2014. Mereka telah bekerja dengan klien dari berbagai bidang seperti industri, makanan, mode, dll. Keberagaman latar belakang dan layanan klien mereka memberi mereka perspektif baru ketika menghadapi tantangan baru.

Mereka menghargai hubungan mereka dengan klien. Pertumbuhan klien mereka sebagai sebuah merek mencerminkan pertumbuhan tim mereka, dan begitu pula sebaliknya. Mereka percaya bahwa pertumbuhan klien mereka dan pertumbuhan mereka sendiri tidak dapat dipisahkan.

## Kontribusi Saya

- Proyek pertama saya adalah aplikasi stok opname dari salah satu toko ritel besar di Surabaya. Aplikasi web ini dapat dijalankan secara offline, dan dapat terhubung ke server lokal selama perangkat terhubung dengan koneksi internet yang sama.
- Proyek kedua saya adalah ERP ringan untuk toko ritel yang sama yang telah saya sebutkan sebelumnya. Sistem ini memiliki banyak fitur seperti pergudangan, pembelian, stok opname (saya memindahkan proyek pertama saya ke sini dengan fitur-fitur yang disempurnakan), dan POS.
- Proyek ketiga saya adalah halaman web untuk memesan kendaraan besar untuk pengiriman. Halaman web ini kemudian disematkan di situs mereka.
- Proyek terakhir saya adalah sistem manajemen proyek yang akan digunakan oleh perusahaan konstruksi. Proyek ini dibuat untuk memenuhi kebutuhan akan sistem manajemen proyek dengan fitur-fitur khusus seperti membuat catatan transfer, mengkategorikan proyek berdasarkan tag dan kategori khusus, serta membuat templat proyek dan tugas untuk karyawan.

## Perjalanan Saya

Saya mengerjakan beberapa proyek di sini. Di sebagian besar proyek, saya menggunakan Laravel sebagai backend dan VueJS sebagai frontend. Awalnya saya hanya menangani backend. Meskipun saya pernah menggunakan Laravel sebelumnya, itu hanya sementara. Dan saat itu, Laravel sudah diperbarui berkali-kali. Jadi, pengetahuan saya saat ini sudah usang. Saya harus belajar Laravel dari awal lagi. :smiling_face_with_tear:

Untungnya dokumentasinya dibuat dengan baik. Saya tidak mengalami kesulitan mengembangkan Laravel di proyek baru. Setelah menyelesaikan backend, saya mulai mengerjakan frontend. Karena ini pertama kalinya saya menggunakan Vue, frontend hampir dikerjakan oleh senior saya dan saya hanya perlu memperbarui konfigurasi untuk API. Saat itu saya belajar tentang VueJS, dan sedikit demi sedikit saya melakukan perubahan hingga akhirnya senior saya mendelegasikan saya sepenuhnya ke frontend. :sunglasses:

Dalam perjalanan saya, saya tidak hanya belajar tentang VueJS tetapi juga tentang TypeScript. Awalnya saya ragu-ragu. Tapi setelah saya mempelajarinya dan menggunakannya di proyek, saya mulai menikmatinya. :laughing: Sampai sekarang, saya selalu menggunakan TypeScript saat mengembangkan menggunakan VueJS.
